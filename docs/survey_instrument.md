## Block 1: Instructions

### Page 1-1

> **Study Instructions**
>
> In this study, you will read **12 scenarios** in which a person is seeking emotional support. For each scenario, you will also see a **response generated by an AI system**.
>
> Some responses may include a small safety indicator (e.g., "✓ Safety reviewed") — this is part of the system design, not a separate task for you.
>
> After reading each scenario and response, you will rate the response on several dimensions using a 7-point scale.
>
> **Important**:
> - There are no right or wrong answers — we are interested in your honest impressions.
> - Please read each scenario carefully before rating.
> - The study takes approximately 20 minutes to complete.

## Block 5: Debriefing

### Page 5-1

> **Thank you for participating!**
>
> **Study purpose**: This study examines how different AI system designs affect people's impressions of emotionally supportive responses. Specifically, we compared three approaches:
>
> - **Single Agent**: A single AI generates a supportive response.
> - **Hidden Checker**: A second AI reviews the response for safety, but the review is not shown to you.
> - **Visible Checker**: A second AI reviews the response for safety, and the review result is shown to you (via a "✓ Safety reviewed" indicator).

## Implementation Notes

### Qualtrics Setup

1. **Randomization**: Use Qualtrics "Randomizer" element within Block 3 to shuffle vignette order.
2. **Embedded data**: Set `cell_id` as embedded data (from Prolific URL parameter or manual assignment). Use display logic to show the correct condition for each vignette per counterbalancing matrix.
3. **Piped text**: Pipe `${e://Field/user_utterance_VXX}` and `${e://Field/response_VXX}` into vignette pages.
4. **Safety indicator**: Use display logic — show the "✓ Safety reviewed" element only when `condition_VXX == C`.
5. **Timing**: Enable "Timing" question on each vignette page to record response latency.
6. **Attention check**: Insert one verification item at a random position (vignette 4 or 8).
7. **Force response**: All Likert items required; open-ended items optional.

### Data Coding

| Variable | Coding |
|----------|--------|
| `cell_id` | 1–12 |
| `vignette_id` | V01–V12 |
| `condition` | A=1 / B=2 / C=3 |
| `risk_level` | low=1 / medium=2 / high=3 |
| `Q1`–`Q8` | 1–7 (Likert) |
| `overall_satisfaction` | 1–7 |
| `crisis_comfort` | 1–7 |
| `mental_workload` | 1–7 |
| `referral_correct` | Count of appropriately checked items (max 7) |
| `response_time_ms` | Milliseconds per vignette page |

### Scoring

- **Empathy Composite** = mean(Q1, Q2)
- **Safety Composite** = mean(Q3, Q4)
- **Trust Composite** = mean(Q6, Q7)
- **Appropriate Reliance Index** = Trust Composite × Q8 (seek-help) as function of risk level
- **Referral Accuracy** = proportion of correctly identified referral-worthy scenarios (items 3–6 in the checklist)
