## Part 1: Post-Interaction Scales

All items rated on a 7-point Likert scale:
**1** = Strongly Disagree → **7** = Strongly Agree

### Perceived Empathy (PE)

| # | Item |
|---|------|
| PE1 | The system seemed to understand how I was feeling. |
| PE2 | The system acknowledged my emotions in a meaningful way. |
| PE3 | The system's responses felt emotionally attuned to my situation. |
| PE4 | I felt that the system genuinely cared about my experience. |

### Perceived Warmth (PW)

| # | Item |
|---|------|
| PW1 | The system felt warm and approachable. |
| PW2 | The system's tone felt natural and human-like. |
| PW3 | The system made me feel comfortable sharing my feelings. |
| PW4 | The system felt cold or mechanical. *(reverse-coded)* |

### Perceived Safety (PS)

| # | Item |
|---|------|
| PS1 | I felt that the system would not give me harmful advice. |
| PS2 | I trusted that the system would recognise when I needed professional help. |
| PS3 | The system seemed to know its own limitations. |
| PS4 | I would feel safe relying on this system during a difficult time. |

### Trust (TR)

| # | Item |
|---|------|
| TR1 | I trust this system to provide appropriate emotional support. |
| TR2 | I believe this system acts in my best interest. |
| TR3 | I would trust this system with sensitive personal information. |
| TR4 | I feel confident in the system's ability to help me. |

### Appropriate Reliance (AR)

| # | Item |
|---|------|
| AR1 | I understand that this system has limitations and is not a replacement for professional help. |
| AR2 | After using this system, I would still seek professional help if my problems persisted. |
| AR3 | This system helped me understand when I might need to talk to a real person. |
| AR4 | I would not depend on this system as my only source of emotional support. |

### Transparency (TP)

*(Conditions B and C only; Condition A sees N/A)*

| # | Item |
|---|------|
| TP1 | I understood how the system decided what to say to me. |
| TP2 | The system was transparent about its decision-making process. |
| TP3 | I appreciate knowing that responses were reviewed for safety. |
| TP4 | The level of explanation provided was appropriate (not too much, not too little). |

### Boundary Clarity (BC)

| # | Item |
|---|------|
| BC1 | The system made it clear that it is not a therapist or medical professional. |
| BC2 | I understood the boundaries of what this system can and cannot do. |
| BC3 | The system appropriately suggested professional help when relevant. |
| BC4 | I felt that the system overstepped its role. *(reverse-coded)* |

### Adoption Intention (AI)

| # | Item |
|---|------|
| AI1 | I would use this system again in the future. |
| AI2 | I would recommend this system to a friend who is going through a tough time. |
| AI3 | I would prefer this system over talking to no one at all. |

## Part 2: Mental Workload (adapted NASA-TLX)

Rate each item from **1** (Very Low) to **7** (Very High):

| # | Dimension | Item |
|---|-----------|------|
| MW1 | Mental Demand | How mentally demanding was it to interact with this system? |
| MW2 | Effort | How hard did you have to work to understand the system's responses? |
| MW3 | Frustration | How frustrated did you feel during the interaction? |

---

## Part 3: Open-Ended Questions

1. **What did you find most helpful about the system's responses?**
2. **What did you find least helpful or concerning about the system's responses?**
3. **Did anything about the system surprise you (positively or negatively)?**
4. **Is there anything you wish the system had done differently?**
5. *(Condition C only)* **How did you feel about seeing the safety review information?**

---

## Part 4: Debrief

*(After all data collected)*

- Reveal the three-condition design
- Explain the maker-checker architecture
- Address any distress caused by the scenarios
- Provide real mental-health resources
